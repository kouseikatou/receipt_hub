# 領収書・請求書の解析

詳細な抽出スキーマは `references/extraction-schema.md` を参照。

## 解析フロー

### 1. ファイル種別の判定
- **PDF**: テキスト抽出を試みる。画像PDFの場合は OCR に切り替え
- **画像 (JPG/PNG/HEIC)**: Gemini Vision で直接解析
- **メール本文**: テキストからパターンマッチで金額・日付を抽出

### 2. 情報抽出
以下の項目を必ず抽出する：

| フィールド | 説明 | 例 |
|----------|------|-----|
| date | 発行日・購入日 | 2024-01-15 |
| vendor | 店名・発行元・先方 | スターバックス渋谷店 |
| amount | 税込金額（円） | 1650 |
| amount_excl_tax | 税抜金額（わかる場合） | 1500 |
| tax_rate | 消費税率 | 10 または 8 |
| doc_type | 種別 | "領収書" または "請求書" |
| category | 勘定科目 | 会議費 |
| memo | 自動生成メモ | "打ち合わせ・スタバ渋谷" |

### 3. 勘定科目の判定
`accounts` スキルの勘定科目ルールを参照して判定する。
確信度が低い場合（候補が2つ以上ある場合）はユーザーに確認を求める。

### 4. 出力形式
```json
{
  "date": "2024-01-15",
  "vendor": "スターバックス渋谷店",
  "amount": 1650,
  "amount_excl_tax": 1500,
  "tax_rate": 10,
  "doc_type": "領収書",
  "category": "会議費",
  "memo": "打ち合わせ・スタバ渋谷",
  "confidence": "high",
  "source": "Gmail",
  "raw_source_id": "thread_xxx"
}
```

## 解析できなかった場合
以下の情報とともに「要確認リスト」に追加する：
- ファイル名・ソース
- 解析を試みた内容
- 判断できなかった理由
