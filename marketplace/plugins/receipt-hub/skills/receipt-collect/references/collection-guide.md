# 収集ガイド

## Gmail 詳細手順

### 検索クエリの組み立て
期間指定が必要な場合:
```
領収書 OR 請求書 OR receipt OR invoice after:2024/01/01 before:2024/01/31
```

添付ファイルのみに絞る場合:
```
(領収書 OR 請求書 OR receipt OR invoice) has:attachment newer_than:30d
```

### メール本文から金額を検出するパターン
- `¥[0-9,]+` または `[0-9,]+円`
- `合計[　 ]*¥?[0-9,]+`
- `金額[　 ]*¥?[0-9,]+`
- `total.*\$?[0-9,]+`

### 重複回避
処理済みのメールには Gmail ラベル「receipt-hub/処理済」を付与する。
次回の収集時はこのラベルがついているものをスキップ。

## Chatwork 詳細手順

### 対象ルームの優先度
1. 「経理」「会計」「finance」を含むルーム名
2. ファイルが添付されているメッセージを含むルーム
3. 全ルーム（件数が多い場合はユーザーに確認）

### ファイル名の判定ルール
以下を含むファイル名を領収書・請求書候補とみなす:
- 領収書、受領書、レシート、receipt
- 請求書、invoice、bill
- 年月日のパターン（例: 20240115, 2024-01-15）

### メッセージ本文の判定
`金額|合計|¥|円|お支払い|ご請求` のいずれかを含むメッセージを候補とする。

## ローカルフォルダ 詳細手順

### スキャンコマンド（PDF・画像）
```bash
find /path/to/folder -type f \( -iname "*.pdf" -o -iname "*.jpg" -o -iname "*.jpeg" -o -iname "*.png" -o -iname "*.heic" \) -newer /path/to/.last_scan
```

### 処理済みファイルの管理
スキャン済みファイルのパスを `.receipt_hub_processed` テキストファイルに記録し、次回スキャン時にスキップする。

## 推奨デフォルトフォルダ構成
```
~/Documents/領収書/
  ├── 未処理/    ← 新しいファイルを置く場所
  ├── 処理済/    ← 処理完了後に移動
  └── 要確認/    ← AI が判断できなかったもの
```
