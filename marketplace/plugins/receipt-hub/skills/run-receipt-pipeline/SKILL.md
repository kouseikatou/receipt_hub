# 領収書パイプライン 一括実行

ユーザーから「領収書処理して」などの指示を受けたら、以下のステップを順番に実行する。

## 実行ステップ

### Step 1: 処理対象の確認
ユーザーに以下を確認する（すでに指定されている場合はスキップ）：
- 対象期間（例：「今月分」「先月分」「○月○日〜○月○日」）
- 記録先スプレッドシートのURL（初回のみ）
- ローカルフォルダのパス（未設定の場合）

### Step 2: 収集フェーズ
`collect-receipts` スキルの手順に従い、3チャネルから領収書・請求書を収集する：
1. Gmail から検索・取得
2. Chatwork から検索・取得
3. ローカルフォルダからスキャン

収集したアイテムをリスト化し、件数をユーザーに報告する。

### Step 3: 解析フェーズ
収集した各アイテムに対して `process-document` スキルの手順を適用する：
- 金額・日付・店名/先方・税込区分・種別（領収書 or 請求書）を抽出
- `accounting-knowledge` スキルを参照して勘定科目を判定
- 判定に自信がない場合は候補を2〜3個提示してユーザーに選ばせる

### Step 4: 確認フェーズ
解析結果をテーブル形式でユーザーに提示する：

```
| 日付 | 店名/先方 | 金額 | 税込 | 勘定科目 | 種別 | メモ |
|------|----------|------|------|---------|------|------|
| ...  | ...      | ...  | ...  | ...     | ...  | ...  |
```

問題がなければスプレッドシートへの書き込みに進む。修正がある場合は対応してから進む。

### Step 5: 記録フェーズ
`sync-to-sheet` スキルの手順に従い、Googleスプレッドシートに記録する。
各行のステータスを「処理済」に更新する。

### Step 6: 完了報告
処理結果を報告する：
- 処理件数（領収書：N件、請求書：N件）
- 合計金額
- スプレッドシートのURL

## 注意事項
- 重複チェックを必ず行う（同じ領収書を2回記録しない）
- 解析できなかったアイテムは「要確認リスト」として別途ユーザーに提示する
- エラーが発生したステップで処理を止め、ユーザーに状況を報告する
