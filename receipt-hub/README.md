# Receipt Hub

Gmail・Chatwork・ローカルフォルダから領収書・請求書を自動収集し、AI解析でGoogleスプレッドシートに記録する経理自動化プラグインです。

## できること

- **マルチチャネル収集**: Gmail・Chatwork・ローカルフォルダを横断して未処理の領収書・請求書を自動収集
- **AI解析**: PDF・画像・メール本文すべてから金額・日付・店名を抽出
- **文脈を汲んだ勘定科目判定**: 「スタバ代 → 会議費」など、内容から適切な科目を自動判定
- **Googleスプレッドシートへの記録**: 標準8列構成で自動記録、freee取り込みにも対応
- **一発実行**: `/run-receipt-pipeline` コマンド一つで全工程を完結

## スキル一覧

| スキル | コマンド | 説明 |
|--------|---------|------|
| 一発実行 | `/run-receipt-pipeline` | 収集→解析→記録を一括実行 |
| 収集 | `/collect-receipts` | 3チャネルから領収書・請求書を収集 |
| 解析 | `/process-document` | 1件の書類をAI解析 |
| 記録 | `/sync-to-sheet` | スプレッドシートに書き込み |
| 勘定科目知識 | 自動 | 日本の勘定科目ルールを参照 |

## 必要な事前設定

### 1. Googleスプレッドシート連携（OAuth方式・サービスアカウント不要）
`skills/sync-to-sheet/references/sheet-setup.md` の手順に従ってセットアップしてください。

概要:
1. `pip3 install gspread` でライブラリをインストール
2. Google Cloud で OAuth クライアントID を作成（デスクトップアプリ）
3. `~/.config/gspread/credentials.json` に保存
4. `python3 -c "import gspread; gspread.oauth()"` を実行 → ブラウザで許可（1回だけ）
5. 以降は完全自動（ブラウザ不要）

### 2. ローカルフォルダの準備（任意）
```
~/Documents/領収書/
  ├── 未処理/    ← 処理したいファイルをここに置く
  └── 処理済/    ← 完了後に自動移動
```

### 3. 接続済みMCPの確認
以下のMCPサーバーがClaudeに接続されていることを確認:
- **Gmail**: メールの検索・閲覧
- **Chatwork**: ルーム・ファイルの取得
- **Google Drive**: ファイルの検索

## スプレッドシートの列構成

| 列 | 内容 |
|----|------|
| A | 日付 |
| B | 店名・先方 |
| C | 金額（税込）|
| D | 税抜金額 |
| E | 消費税率 |
| F | 勘定科目 |
| G | 種別（領収書/請求書）|
| H | メモ |
| I | ソース（Gmail/Chatwork/ローカル）|
| J | ステータス（未処理/処理済）|
| K | 記録日時 |
